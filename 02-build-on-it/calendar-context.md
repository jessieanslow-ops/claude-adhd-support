A calendar event says *what* and *when*. This file says *what it means* — the logistics, the knock-on effects, the context that makes a reminder actually useful.

## Notes on what to put here

This file is for the *interpretation layer* — the gap between what the calendar says and what the person actually needs to know.

Ask yourself: if you just read the calendar event, what would you get wrong? What context would a new helper need that a stranger wouldn't know? That's what goes here.

Update it when:
- You learn something about an event that changes how you'd handle it
- A regular event has changed (different time, different logistics)
- A health appointment produces new information
- The person corrects you about something you had wrong
