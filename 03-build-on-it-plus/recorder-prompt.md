# Recorder prompt

Run this at the end of each day (default: 21:00–22:00 local time, or before any context clear).

This is your instruction for reviewing what happened and updating the files that carry context forward. The goal is to make tomorrow's session smarter than today's — not to log everything, but to capture what changed.

---

## What to review

Read today's conversation log (or your session notes). Look for:

- Things you learned about how this person works that you didn't know before
- Corrections — moments where they told you something was wrong
- Patterns that held (or broke)
- Loops that progressed, stalled, or completed
- Energy level and how it tracked against predictions
- Anything that should surface tomorrow or this week

---

## What to update

### patterns.md

**Working memory (48h):** Write a brief note about today — energy, what happened, anything notable. Prune entries older than 48 hours from this section before adding.

**Weekly patterns:** Update only if something new confirmed or contradicted an existing pattern. Don't add observations until you've seen them twice.

**Monthly insights:** Don't touch unless something major shifted (a new rhythm established, a change in circumstances).

**Regression gate:** Before saving, check that nothing with context has been removed without being superseded. If an important observation would disappear, either keep it or record why it's been replaced.

### todo.md

- Move completed items to Done section
- Update loop ages (days since added)
- Escalate anything that's crossed a threshold (3-6d → nudge zone, 7-13d → flagged, 14d+ → force decision next session)
- Prune Done section to items completed in the last 7 days

### next_session.md (orientation header)

Update the top section with:
- What happened today (2–3 lines, not a log)
- What's in flight that needs picking up tomorrow
- Anything with a deadline in the next 48 hours

Keep this short. It's a handoff note, not a diary entry. The person reading it is you, tomorrow, without today's context.

### craft-and-learnings.md

Update only if something changed. A correction deserves an entry. A pattern you've noticed in how they communicate deserves an entry. Small things compound.

---

## What not to do

- Don't log everything — the goal is signal, not a transcript
- Don't move something to Done until it's actually done (not just responded to)
- Don't add a pattern from a single observation — wait for it to repeat
- Don't update next_session.md with so much detail that it's overwhelming to read tomorrow

---

## After updating

Review next_session.md and check: if you woke up tomorrow with only that file and patterns.md, would you have enough to continue well? If no, add what's missing. If yes, stop.
